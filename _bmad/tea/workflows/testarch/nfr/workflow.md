# Workflow: NFR-тестирование (Non-Functional Requirements)

## Purpose
Верификация нефункциональных требований: производительность, безопасность, доступность, надёжность, масштабируемость.

## Input
- NFR-спецификация (SLA: время ответа, throughput, uptime)
- Архитектурные требования и ограничения

## Steps

### Step 1: Categorize (Категоризация)
- Действия: определить NFR-атрибуты для тестирования
- Категории:
  - **Performance**: время ответа, пропускная способность
  - **Security**: OWASP Top 10, аутентификация, авторизация
  - **Accessibility**: WCAG 2.1/2.2
  - **Reliability**: отказоустойчивость, recovery
  - **Scalability**: горизонтальное/вертикальное масштабирование
- Результат: список NFR с приоритетами

### Step 2: Define Metrics (Определение метрик)
- Действия: для каждого NFR — конкретный измеримый критерий
- Примеры: p95 latency < 200ms, uptime 99.9%, Lighthouse score > 90
- Результат: NFR checklist с целевыми значениями

### Step 3: Select Tools (Выбор инструментов)
- Performance: k6, JMeter, Locust, Lighthouse
- Security: OWASP ZAP, Burp Suite, SonarQube
- Accessibility: axe-core, Lighthouse, WAVE
- Результат: выбранные инструменты, описание настройки

### Step 4: Automate Scripts (Автоматизация)
- Действия: написать сценарии NFR-тестов; интегрировать в CI
- Для performance: profile тесты (короткие) + нагрузочные (ночные)
- Результат: запускаемые NFR-тесты

### Step 5: Baseline & Tresholds (Базовый уровень)
- Действия: установить baseline первого прогона; определить пороги нарушений
- Результат: пороги, при которых пайплайн падает (fail/pass)

### Step 6: Report (Отчёт)
- Действия: визуализировать метрики (графики, тренды); сравнить с baseline
- Результат: NFR-дашборд, отчёт для команды и стейкхолдеров

## Output
- Набор автоматизированных NFR-тестов
- Метрики, baseline и пороговые значения
- Дашборд с историей изменений NFR-показателей
