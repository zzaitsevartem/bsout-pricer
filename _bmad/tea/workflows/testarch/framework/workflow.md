# Workflow: Выбор и настройка тестового фреймворка

## Purpose
Выбор оптимального тестового фреймворка под проект, его настройка и документирование соглашений.

## Input
- Тип проекта (web / mobile / API / desktop)
- Стек технологий (JS/TS, Python, Java, C#)
- Требования: CI, отчёты, параллельный запуск, поддерживаемость

## Steps

### Step 1: Requirements (Сбор требований)
- Действия: определить язык, среду запуска, CI-систему, необходимые интеграции
- Критерии: скорость, стабильность, документация, комьюнити
- Результат: список требований к фреймворку

### Step 2: Evaluate (Оценка)
- Действия: сравнить 2-3 кандидата по требованиям
- Для web E2E: Playwright / Cypress / Selenium
- Для API: Supertest / REST Assured / Postman Newman
- Для unit: Jest / PyTest / NUnit / JUnit
- Результат: выбранный фреймворк с обоснованием

### Step 3: Setup (Настройка)
- Действия: инициализация проекта, настройка конфигурации, подключение репортера
- Пример: `npm init playwright`, Allure, GitHub Actions
- Результат: рабочий скелет тестового проекта

### Step 4: Conventions (Соглашения)
- Действия: описать структуру директорий, именование, page objects, fixtures
- Артефакт: CONTRIBUTING.md / testing-conventions.md
- Результат: единые правила для команды

### Step 5: Pilot (Пилот)
- Действия: написать 5-10 тестов на реальных сценариях; прогнать в CI
- Результат: проверка гипотезы, корректировка при необходимости

## Output
- Выбранный и настроенный тестовый фреймворк
- Документация соглашений и структуры проекта
- Набор пилотных тестов, интегрированных в CI
