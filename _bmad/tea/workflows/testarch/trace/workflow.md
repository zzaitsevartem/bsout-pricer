# Workflow: Трассировка (Traceability)

## Purpose
Обеспечение прослеживаемости: каждый тест связан с требованием, каждый requirement покрыт тестом.

## Input
- Требования (user stories, функциональные требования)
- Тест-кейсы / автотесты

## Steps

### Step 1: Define Traceability Matrix (Создание матрицы)
- Действия: создать матрицу трассировки (requirements × test cases)
- Формат: таблица (Google Sheets, Notion, или в коде — YAML-маппинг)
- Типы связей: covers (покрывает), relates-to (связан), validates (валидирует)
- Результат: Traceability Matrix (RTM)

### Step 2: Map Requirements (Мэппинг требований)
- Действия: каждому requirement сопоставить ID тест-кейса
- Ссылки в коде: `@req-id` в аннотации теста
- Результат: установленные связи requirement ↔ test

### Step 3: Validate Coverage (Валидация покрытия)
- Действия: найти requirement без тестов (gaps) и тесты без requirement (orphans)
- Отчёт: % coverage требований тестами
- Результат: анализ покрытия с визуализацией

### Step 4: Close Gaps (Закрытие пробелов)
- Действия: для uncovered requirements — написать тесты
- Для orphan tests — определить, какой requirement они покрывают, или удалить
- Результат: полное покрытие требований тестами

### Step 5: Maintain (Поддержка)
- Действия: обновлять матрицу при изменении требований
- Автоматизация: CI check при мёрже — проверять coverage требований
- Результат: актуальная трассировка на каждом релизе

## Output
- Матрица трассировки (requirements ↔ tests)
- Метрика покрытия требований тестами
- Процесс автоматической проверки трассировки в CI
