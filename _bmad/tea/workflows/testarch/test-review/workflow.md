# Workflow: Ревью тестов (Test Review)

## Purpose
Обеспечение качества тестового набора через структурированное ревью коллегами.

## Input
- Тест-кейсы / автотесты на ревью
- Требования / user story

## Steps

### Step 1: Self-Check (Самопроверка)
- Действия: автор проверяет тесты перед отправкой на ревью
- Чек-лист:
  - Тесты запускаются и проходят
  - Нет flaky-тестов
  - Код читаемый, нет хардкода
  - Покрыты happy path + error + граничные случаи
- Результат: готовность к ревью

### Step 2: Assign (Назначение)
- Действия: назначить ревьюера (знакомого с функционалом)
- Договориться о сроках ревью
- Результат: ревьюер назначен

### Step 3: Review (Ревью)
- Фокус:
  - **Correctness** — тест проверяет то, что заявлено
  - **Coverage** — ключевые сценарии покрыты
  - **Readability** — понятные шаги, осмысленные assert'ы
  - **Maintainability** — нет дублирования, page objects используются
  - **Independence** — тесты не зависят друг от друга
- Результат: комментарии, approve / request changes

### Step 4: Fix (Доработка)
- Действия: автор исправляет замечания
- Повторное ревью при необходимости
- Результат: финальная версия тестов

### Step 5: Merge (Вливание)
- Действия: после approve — мёрж в основную ветку
- Результат: тесты в кодовой базе, интегрированные в CI

## Output
- Проверенные и утверждённые тесты
- Метрики ревью: количество замечаний, время ревью
- Улучшение качества тестового кода в команде
