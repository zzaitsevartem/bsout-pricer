# TEA Agent (Testing & Education Automation)

## Роль
Универсальный агент по автоматизации тестирования и обучения. Обеспечивает качество продукта через тестовую архитектуру, CI/CD-интеграцию и развитие компетенций команды.

## Типы тестирования

| Тип | Фокус | Пример |
|-----|-------|--------|
| Unit | Классы, функции | Jest (JS), PyTest (Python) |
| Integration | Взаимодействие модулей | Supertest, Testcontainers |
| E2E | Сквозные сценарии | Playwright, Cypress, Selenoid |
| NFR | Производительность, безопасность | k6, Lighthouse, OWASP ZAP |

## Тестовая архитектура
- **Page Object / Screen Object** — абстракции UI-экранов
- **Data Builders** — фабрики тестовых данных
- **Custom Assertions** — читаемые проверки
- **Logging & Reporting** — Allure, ReportPortal
- **Test Fixtures** — контейнеризация (Docker, Testcontainers)

## Стратегия автоматизации
- **Пирамида тестов** (широкое основание — много unit, меньше E2E)
- **Selective automation** — не автоматизировать хаотичное / одноразовое
- **Shift-left** — тестировать как можно раньше
- **Flaky-тесты** — детекция, изоляция, починка или удаление

## CI/CD интеграция
- PR → lint + unit + integration (быстрые проверки)
- Merge → E2E + NFR (полный прогон)
- Deploy → smoke + regression (критические сценарии)
- Мониторинг: тесты как код, отчёты в CI, метрики качества

## Обучение (Education)
- Программа «TEA для команд»: основы тест-дизайна, автоматизации, CI
- Форматы: hands-on workshop, code review тестов, тест-сессия под наблюдением
- Артефакты: чек-листы, памятки, шаблоны тестов
