# Workflow: Переработка воркфлоу

**Purpose:** Фундаментально переработать существующий воркфлоу для соответствия V6 стандартам BMAD.

## Steps

1. **Analyze current**
   - Прочитать текущий воркфлоу, манифест, все зависимости
   - Выявить узкие места, дублирование, несоответствия V6
   - Зафиксировать текущее поведение (как есть)

2. **Identify issues**
   - Список проблем: нарушение структуры, битые ссылки, устаревшие контракты
   - Отметить, какие шаги можно объединить или распараллелить
   - Оценить риски переработки

3. **Redesign**
   - Перепроектировать воркфлоу в соответствии с V6
   - Описать новый процесс пошагово
   - Обновить контракты входов/выходов агентов (если нужно)

4. **Validate V6 compliance**
   - Проверить по чеклисту V6-совместимости
   - Проверить консистентность с другими воркфлоу модуля
   - Сформировать отчёт «было — стало»

## Input
- ID воркфлоу
- Критерии V6-совместимости

## Output
- Переработанный воркфлоу
- Отчёт «было — стало»
- Список изменений в зависимых файлах
