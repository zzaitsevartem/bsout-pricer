# Workflow: Редактирование воркфлоу

**Purpose:** Внести изменения в существующий BMAD-воркфлоу.

## Steps

1. **Review**
   - Прочитать текущий воркфлоу, понять его цель и шаги
   - Найти все ссылки на этот воркфлоу из других файлов

2. **Plan**
   - Определить, какие шаги добавляются / изменяются / удаляются
   - Оценить влияние изменения на контракты агентов
   - Если изменения ломают обратную совместимость — уведомить пользователей

3. **Implement**
   - Внести изменения в .md-файл воркфлоу
   - Обновить связанные воркфлоу (при необходимости)
   - Обновить README модуля (если changed)

4. **Validate**
   - Проверить, что воркфлоу остался консистентным
   - Проверить ссылки на агентов
   - Убедиться, что последовательность шагов логична

## Input
- ID воркфлоу
- Описание изменений

## Output
- Обновлённый файл воркфлоу
- Обновлённые зависимые файлы
