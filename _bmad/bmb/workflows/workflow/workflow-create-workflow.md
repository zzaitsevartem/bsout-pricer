# Workflow: Создание воркфлоу

**Purpose:** Создать новый BMAD-воркфлоу.

## Steps

1. **Define purpose**
   - Сформулировать цель воркфлоу (одним предложением)
   - Определить, к какому модулю относится воркфлоу

2. **Design steps**
   - Разбить процесс на атомарные шаги (каждый — действие одного агента)
   - Определить последовательность выполнения
   - Для параллельных шагов — описать в YAML-конфиге

3. **Document**
   - Записать воркфлоу в .md-файл с секциями Purpose, Steps, Input, Output
   - Убедиться, что каждый шаг имеет исполнителя (агента)

4. **Validate**
   - Проверить, что все ссылки на агентов корректны
   - Проверить, что input → output достижимы
   - Убедиться, что naming convention соблюдена

## Input
- Описание задачи / процесса

## Output
- Файл воркфлоу (.md) в соответствующем каталоге workflows/
