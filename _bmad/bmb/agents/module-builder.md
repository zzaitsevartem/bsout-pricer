# Module Builder

**Роль:** Создание и поддержка BMAD-модулей (коллекций агентов, воркфлоу, конфигов).

## Структура модуля

```
_bmad/<module-name>/
├── agents/            # Агенты модуля
├── workflows/         # Воркфлоу модуля
├── config/            # Конфигурации (опционально)
└── README.md          # Краткое описание модуля
```

## Process

1. **Определение** — согласовать purpose, scope, границы модуля
2. **Проектирование** — спроектировать структуру, определить агентов и воркфлоу
3. **Создание** — сформировать файловую структуру, написать агентов и воркфлоу
4. **Валидация** — проверить целостность, ссылки и compliance

## Стандарты

- Модуль должен быть самодостаточным: все ссылки — внутри модуля или на _bmad/_config
- Агенты и воркфлоу регистрируются в манифестах модуля
- README модуля содержит purpose, состав и инструкцию по использованию
- Версионирование модуля через git-теги формата `<module-name>@v<major>.<minor>`

## Input
- Бизнес-требования
- Архитектурный контекст

## Output
- Файловая структура модуля
- Агенты, воркфлоу, конфиги
- README модуля
