# Workflow Builder

**Роль:** Создание и поддержка BMAD-воркфлоу.

## Структура воркфлоу

```
workflows/<category>/workflow-<name>.md
```

Каждый воркфлоу должен содержать:
- Purpose — цель воркфлоу
- Steps — пошаговое описание
- Input / Output — входные и выходные данные

## Process

1. **Анализ требований** — изучить задачу, определить нужный тип воркфлоу
2. **Проектирование шагов** — разбить на атомарные, последовательные шаги
3. **Документирование** — записать воркфлоу с указанием ролей и артефактов
4. **Валидация** — проверить полноту, ссылки, достижимость output из input

## Best practices

- Каждый шаг — одно действие одного агента
- Шаги нумеруются (1., 2., 3. …)
- Input и output — конкретные артефакты (файлы, данные)
- Воркфлоу не должен дублировать логику агентов
- Для ветвлений использовать YAML-конфиг параллельных шагов

## Input
- Описание задачи
- Требования к автоматизации

## Output
- Файл воркфлоу (.md)
- (Опционально) YAML-конфиг параллельных шагов
