# Workflow: Создание Product Brief

## Назначение
Сформировать краткое описание продукта (Product Brief) через коллаборативный discovery-процесс для согласования видения и границ продукта.

## Входные данные
- Бизнес-гипотеза или идея продукта
- Результаты первичных исследований (при наличии)
- Список стейкхолдеров

## Участники
- Product Manager (владелец процесса)
- Tech Lead
- UX-аналитик
- Ключевые стейкхолдеры

## Шаги

### 1. Определение проблемы
- Сформулировать core problem statement
- Задокументировать контекст возникновения проблемы
- Собрать качественные/количественные подтверждения

### 2. Целевая аудитория
- Определить сегменты пользователей
- Описать User Persona (роль, цели, боли)
- Указать размер сегмента и охват

### 3. Ценностное предложение
- Сформулировать уникальное ценностное предложение (UVP)
- Описать сценарий использования "как есть" и "как будет"
- Определить ключевые функции MVP

### 4. Метрики успеха
- Определить North Star Metric
- Выбрать leading/lagging индикаторы
- Установить целевые значения для первого релиза

### 5. Границы и ограничения
- Определить scope первого релиза (MVP)
- Задокументировать out-of-scope
- Указать технические/бизнес-ограничения

## Выходные данные
- Документ Product Brief
- Согласованное видение продукта со стейкхолдерами
- Список открытых вопросов на следующий этап
