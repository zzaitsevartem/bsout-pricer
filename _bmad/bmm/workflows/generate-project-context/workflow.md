# Workflow: Генерация project-context.md

## Назначение
Создать или обновить файл project-context.md с правилами для AI-агентов, описывающий структуру, архитектуру и конвенции проекта.

## Входные данные
- AGENTS.md (корневой)
- Структура каталогов проекта
- Ключевые конфигурационные файлы
- Существующий project-context.md (при наличии)

## Участники
- Tech Lead (инициатор)
- Архитектор
- AI-агент (исполнитель генерации)

## Шаги

### 1. Сбор информации о проекте
- Прочитать AGENTS.md и другие корневые инструкции
- Изучить структуру каталогов (clients, services, _bmad)
- Определить ключевые технологии и версии

### 2. Анализ архитектуры
- Изучить архитектурные документы
- Выделить ключевые модули и их назначение
- Зафиксировать паттерны и конвенции

### 3. Определение правил для AI
- Сформулировать правила код-стайла
- Указать обязательные шаги перед изменениями
- Определить требования к тестам и документации

### 4. Генерация файла
- Сформировать project-context.md по шаблону
- Добавить секции: overview, structure, rules, architecture
- Пронумеровать правила для ссылок

### 5. Ревью и утверждение
- Показать Tech Lead на ревью
- Внести правки по замечаниям
- Разместить в корне проекта

## Выходные данные
- project-context.md
- Обновлённые рекомендации для AI-агентов
