# Workflow: Декомпозиция на эпики и User Stories

## Назначение
Разбить требования PRD на управляемые эпики и user stories, готовые для планирования спринтов и передачи в разработку.

## Входные данные
- PRD (утверждённый)
- Архитектурный документ
- UX-макеты и прототипы

## Участники
- Product Manager
- Tech Lead
- Команда разработки

## Шаги

### 1. Формирование эпиков
- Сгруппировать требования по функциональным областям
- Определить границы каждого эпика
- Назначить владельца эпика

### 2. Декомпозиция эпиков в stories
- Разбить эпик на атомарные user stories
- Каждая story: одна пользовательская ценность
- Добавить acceptance criteria в BDD-формате

### 3. Технические задачи
- Выделить инфраструктурные и технические истории
- Добавить задачи по миграции данных, настройке CI/CD
- Оценить технический долг

### 4. Оценка и приоритизация
- Оценить stories в story points / часах
- Расставить приоритеты в рамках эпика
- Сформировать предварительный roadmap

### 5. Валидация готовности
- Проверить, что каждая story имеет: описание, AC, оценку
- Согласовать с командой на backlog grooming
- Утвердить Definition of Ready

## Выходные данные
- Список эпиков с описанием границ
- Детализированный Product Backlog
- Roadmap по эпикам
- Definition of Ready для stories
