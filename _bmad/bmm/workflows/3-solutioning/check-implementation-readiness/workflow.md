# Workflow: Проверка готовности к реализации

## Назначение
Валидировать, что все необходимые артефакты (PRD, UX, архитектура, эпики) полны, согласованы и готовы для передачи в разработку.

## Входные данные
- PRD
- Архитектурный документ
- UX-дизайн (макеты, прототипы)
- Product Backlog (эпики и stories)
- Чеклист готовности

## Участники
- Product Manager
- Tech Lead
- Архитектор
- QA-инженер

## Шаги

### 1. Проверка PRD
- Все функциональные требования описаны
- Приоритеты расставлены
- Нет противоречий с UX и архитектурой

### 2. Проверка UX
- Все ключевые экраны спроектированы
- User flows покрывают все user stories
- Дизайн-спецификации переданы разработке

### 3. Проверка архитектуры
- ADR зафиксированы
- API-контракты определены
- Интеграции описаны
- NFR учтены

### 4. Проверка эпиков и stories
- Backlog приоритизирован
- Каждая story имеет acceptance criteria
- Оценки проставлены
- Definition of Ready выполнен

### 5. Итоговое заключение
- Pass: артефакты готовы к реализации
- Fail: указать, что именно требуется доработать
- Назначить ответственных за доработку

## Выходные данные
- Отчёт о готовности к реализации
- Список blocker-ов (если есть)
- Решение: Start / Blocked
