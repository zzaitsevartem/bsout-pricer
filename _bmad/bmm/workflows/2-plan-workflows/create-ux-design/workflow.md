# Workflow: Проектирование UX-дизайна

## Назначение
Разработать пользовательский опыт продукта: user flows, вайрфреймы, прототипы, руководство по дизайн-системе.

## Входные данные
- PRD с user stories
- Product Brief
- Исследования пользователей
- Дизайн-система проекта (если существует)

## Участники
- UX-дизайнер
- Product Manager
- Tech Lead
- UI-дизайнер

## Шаги

### 1. User Flows
- Построить CJM (Customer Journey Map) для ключевых сценариев
- Определить точки входа, шаги, точки принятия решений
- Согласовать flows с Product Manager

### 2. Вайрфреймы (Low-fidelity)
- Нарисовать каркасы ключевых экранов
- Показать навигацию между экранами
- Проверить соответствие user stories

### 3. Дизайн-система и компоненты
- Определить необходимые UI-компоненты
- Следовать существующей дизайн-системе
- При необходимости — расширить компонентную базу

### 4. High-fidelity прототип
- Отрисовать финальные макеты
- Собрать интерактивный прототип (Figma/Sketch)
- Провести юзабилити-тестирование

### 5. Передача в разработку
- Подготовить UI Kit / design specs
- Добавить asset-ы и иконки
- Провести handoff-митинг с командой

## Выходные данные
- User Flow диаграммы
- Вайрфреймы / High-fidelity макеты
- Интерактивный прототип
- Design Specs для разработки
