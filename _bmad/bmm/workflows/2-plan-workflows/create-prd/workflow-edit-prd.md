# Workflow: Редактирование существующего PRD

## Назначение
Внести контролируемые изменения в утверждённый PRD-документ с сохранением истории версий и согласованием изменений.

## Входные данные
- Текущая версия PRD
- Запрос на изменение (Change Request)
- Обоснование изменений

## Участники
- Product Manager (автор изменений)
- Tech Lead
- QA-инженер

## Шаги

### 1. Анализ запроса
- Понять причину изменения: новая информация, баг, изменение рынка
- Оценить impact на scope, сроки, ресурсы
- Задокументировать rationale

### 2. Редактирование
- Открыть текущую версию PRD
- Внести изменения с пометкой версии
- Обновить affected user stories и AC

### 3. Оценка влияния
- Проверить связанные артефакты (UX, архитектура)
- Обновить эпики и stories в бэклоге
- Пересчитать приоритеты при необходимости

### 4. Ревью и утверждение
- Направить PRD на ревью ключевым участникам
- Получить approval
- Зафиксировать версию и changelog

## Выходные данные
- Новая версия PRD
- Changelog изменений
- Обновлённый Product Backlog
