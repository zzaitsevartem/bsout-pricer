# Workflow: Валидация PRD

## Назначение
Проверить PRD-документ на соответствие стандартам качества, полноту и консистентность перед передачей в разработку.

## Входные данные
- PRD-документ (любая версия)
- Чеклист валидации
- Стандарты оформления требований

## Участники
- Product Manager (исполнитель)
- QA-инженер (валидатор)
- Архитектор (проверка реализуемости)

## Шаги

### 1. Проверка структуры
- Наличие всех обязательных разделов
- Консистентность терминологии
- Корректность форматирования

### 2. Валидация требований
- Каждая user story имеет acceptance criteria
- AC написаны в тестируемом формате (Given/When/Then)
- Нет противоречий между требованиями
- Требования атомарны и независимы

### 3. Проверка реализуемости
- Техническая выполнимость (feasibility check)
- Соответствие архитектурным решениям
- Оценка трудозатрат присутствует

### 4. Полнота покрытия
- Все сценарии из Product Brief покрыты
- Учтены нефункциональные требования
- Описаны граничные случаи и error-handling

### 5. Итоговый вердикт
- Pass: PRD соответствует стандартам
- Pass with comments: мелкие замечания
- Fail: требуется доработка

## Выходные данные
- Отчёт валидации
- Список замечаний и рекомендаций
- Статус: Approved / Needs Revision
