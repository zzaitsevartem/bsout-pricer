# Workflow: Создание PRD

## Назначение
Разработать документ требований к продукту (PRD) на основе результатов discovery-фазы: функциональные требования, user stories и критерии приёмки.

## Входные данные
- Product Brief
- Результаты исследований (market, domain, technical)
- Бизнес-цели и OKR

## Участники
- Product Manager (автор)
- Tech Lead (валидация реализуемости)
- UX-дизайнер
- QA-инженер

## Шаги

### 1. Сбор входных данных
- Систематизировать артефакты discovery-фазы
- Провести kickoff с командой
- Согласовать scope версии

### 2. Определение функциональных требований
- Описать функции по эпикам
- Для каждой функции: цель, триггер, результат
- Приоритизировать по MoSCoW

### 3. Формирование User Stories
- Написать user stories по шаблону
- Добавить acceptance criteria (BDD-формат)
- Учесть негативные сценарии

### 4. Приоритизация и бэклог
- Оценить бизнес-ценность и усилия
- Упорядочить backlog
- Определить MVP и следующие релизы

### 5. Написание и ревью PRD
- Оформить PRD-документ
- Провести ревью с командой и стейкхолдерами
- Утвердить и зафиксировать версию

## Выходные данные
- PRD-документ (утверждённый)
- Приоритизированный Product Backlog
- Список открытых вопросов и рисков
