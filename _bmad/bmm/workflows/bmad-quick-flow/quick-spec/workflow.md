# Workflow: Быстрая спецификация (Quick Spec)

## Назначение
Создать минимальную спецификацию для небольших изменений (багфиксы, мелкие фичи, технические доработки) без полного цикла PRD.

## Входные данные
- Описание проблемы или запрос на изменение
- Контекст продукта
- Существующие артефакты (при наличии)

## Участники
- Product Manager / Tech Lead
- Разработчик

## Шаги

### 1. Формулировка проблемы
- Описать проблему или требуемое изменение
- Приложить скриншоты, логи, ссылки
- Указать окружение и шаги воспроизведения (для багов)

### 2. Предлагаемое решение
- Описать подход к реализации
- Оценить затрагиваемые модули и код
- Указать риски и альтернативы

### 3. Acceptance Criteria
- Описать критерии в формате Given/When/Then
- Указать негативные сценарии
- Определить границы решения

### 4. Оценка трудозатрат
- Оценить время реализации
- Указать зависимости и блокеры
- Определить приоритет

## Выходные данные
- Quick Spec документ
- Согласованное решение
- Задача в бэклоге
