# Workflow: Быстрая реализация (Quick Dev)

## Назначение
Реализовать небольшое изменение из Quick Spec без полного цикла разработки: одна ветка, короткое ревью, быстрый релиз.

## Входные данные
- Quick Spec документ
- Репозиторий проекта
- Dev-среда

## Участники
- Разработчик (исполнитель)
- Tech Lead (ревью)

## Шаги

### 1. Подготовка
- Убедиться, что Quick Spec утверждён
- Создать feature-ветку от main
- Ознакомиться с affected кодом

### 2. Реализация
- Написать код минимальными изменениями
- Следовать существующим паттернам
- Добавить тесты (как минимум на ключевую логику)

### 3. Самопроверка
- Запустить линтер и тесты
- Проверить на dev-стенде
- Убедиться в отсутствии регрессии

### 4. Быстрое ревью
- Открыть MR с коротким описанием
- Tech Lead проводит ускоренное ревью
- Исправить замечания (если есть)

### 5. Релиз
- Смержить ветку
- Развернуть на staging/production
- Закрыть задачу в трекере

## Выходные данные
- Закоммиченный код в main
- Пройденные тесты
- Развёрнутое изменение
