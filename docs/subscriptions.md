# Подписки и тарифы BScout

## Общая логика

Платформа работает с **5 магазинами** города Ставрополя: ТГСМ, Профи, Либерти, Гринспарк, Дивизион. Количество магазинов **фиксировано** и одинаково для всех тарифов — ограничений по магазинам нет.

Различия между тарифами касаются:
- **Умного поиска (fuzzy)** — эксклюзив продвинутого
- **Уведомлений о падении цен** — эксклюзив продвинутого
- **Количества отслеживаемых товаров**
- **Экспорта отчётов**
- **Уровня поддержки**

## Целевая аудитория и её потребности

### Сервисный центр (основной клиент)
- Ежедневно ищет десятки запчастей: дисплеи, аккумуляторы, шлейфы, контроллеры
- **Критично:** скорость подбора, минимальная закупочная цена
- **Поведение:** знает приблизительное название, часто сталкивается с тем, что магазины называют одну деталь по-разному
- **Боль:** тратит время на обзвон магазинов, когда поиск не находит нужное

### Частный мастер
- Регулярно заказывает запчасти для ремонта
- **Критично:** экономия времени и денег
- **Поведение:** следит за ценами на популярные детали, ждёт снижений
- **Боль:** упускает выгодные предложения, потому что не отслеживает цены

### Обычный пользователь
- Разово ремонтирует телефон/ноутбук
- **Критично:** простота, не переплатить
- **Поведение:** может не знать точного названия детали
- **Боль:** не может найти нужную запчасть, сдаётся и идёт в магазин

### 1. Пробный (Trial)

| Параметр | Значение |
|----------|----------|
| Стоимость | 0 ₽ |
| Длительность | 7 календарных дней |
| Магазины | все 5 |
| Поиск | точное + частичное совпадение (ILIKE) |
| Мониторинг товаров | до 10 |
| Уведомления о снижении цен | — |
| Экспорт отчётов | — |
| Умный поиск (fuzzy) | — |
| Поддержка | — |

Пробный период **полнофункционален** — включает всё, что доступно на базовом тарифе. Единственное ограничение — время (7 дней) и количество отслеживаемых товаров (10).

**Что происходит после 7 дней:**
- Поиск **блокируется** до оформления подписки
- История поиска и данные **сохраняются**
- При оформлении подписки в течение 7 дней после истечения — скидка 20% на первый платёж

### 3. Продвинутый (Advanced)

| Параметр | Значение |
|----------|----------|
| Стоимость | 499 ₽/мес |
| Магазины | все 5 |
| Поиск | точное + частичное + **умный (fuzzy)** |
| Мониторинг товаров | до 500 |
| Уведомления о снижении цен | ✓ |
| Экспорт отчётов | PDF, CSV |
| Умный поиск (fuzzy) | ✓ |
| Поддержка | 24/7 |

**Умный поиск (fuzzy)** — ключевая фишка. Магазины называют одну деталь по-разному: «Дисплей iPhone 13», «Экран инцел iPhone 13», «LCD iPhone 13». Умный поиск находит всё это одним запросом. Работает при опечатках, транслитерации, неправильной раскладке:
- "dsiplay iphone 13" → находит «Дисплей iPhone 13»
- "экран Самсунг Галакси С22" → находит «Дисплей Samsung Galaxy S22 Ultra»

**Уведомления о снижении цен** — отслеживаешь 500 товаров, а система сама сообщает, когда цена падает. Купил дисплей за 4500 ₽, а через неделю он стоит 3800 — ты узнаешь об этом первым.

**Экспорт отчётов** — PDF и CSV для бухгалтерии, отчётности, анализа закупок.

**500 отслеживаемых товаров** — весь ассортимент сервисного центра под контролем.

**Поддержка 24/7** — сломался поиск в 2 часа ночи перед срочным заказом? Помощь приходит сразу, а не завтра утром.

**Для кого:** активные сервисные центры, частные мастера с постоянным потоком заказов.

## Почему именно так

### Умный поиск — это не «nice to have»
Для мастера, который ищет «дисплей инцел самсунг а53», а магазин называет его «LCD-модуль Samsung Galaxy A53» — умный поиск экономит 10–15 минут на каждый запрос. При 30 запросах в день это 5+ часов в неделю.

### Уведомления о цене — прямая экономия
Мастера закупают одни и те же детали (дисплеи, аккумуляторы, шлейфы) постоянно. Если цена на дисплей iPhone 13 упала на 500 ₽ — мастер должен узнать об этом. На базовом тарифе он узнает случайно, зайдя на сайт. На продвинутом — сразу.

### Экспорт — нужен для бухгалтерии
Сервисные центры — ИП и ООО. Им нужен учёт закупок. PDF/CSV отчёты позволяют не копировать цены вручную.

### API-доступ исключён из тарифов
Целевая аудитория — мастера без разработчиков. Работают через браузер. API может появиться позже как B2B-фича.

### Разница в 100 ₽ — намеренно минимальная
100 ₽ разницы при добавлении умного поиска + уведомлений + экспорта + 24/7 поддержки — очевидная выгода. Мастер, который экономит 500 ₽ на одном дисплейном заказе благодаря уведомлению, уже окупил разницу.

## Lifecycle подписки

```
Регистрация
    │
    ▼
Пробный период (7 дней, до 10 товаров)
    │
    ├── Оформил подписку → Базовый / Продвинутый
    │       │
    │       ├── Автоплатёж (ежемесячно)
    │       ├── Ручное продление
    │       └── Отмена → доступ до конца периода → блокировка
    │
    └── Не оформил → Блокировка поиска
            │
            ├── Оформил в течение 7 дней → скидка 20%
            └── Позже → полная стоимость
```

### Смена тарифа

- **Базовый → Продвинутый:** мгновенный апгрейд. Стоимость перерасчитывается пропорционально оставшимся дням текущего периода.
- **Продвинутый → Базовый:** downgrade вступает в силу после окончания текущего оплаченного периода.
- **Любой → Отмена:** доступ сохраняется до конца текущего оплаченного периода. После — блокировка поиска, данные сохраняются.

### Возврат средств

- Полный возврат — в течение 3 дней после оплаты, если подписка не использовалась.
- Далее — без возврата, доступ до конца периода.

## Скидки и промо

- **Первая оплата:** скидка 20% на любой платный тариф.
- **Скидка за просрочку:** если пользователь оформляет подписку в течение 7 дней после окончания пробного периода — скидка 20% (суммируется с приветственной не более 20%).
- **Промокоды:** возможность ввода промокода при оформлении подписки (админ генерирует в админ-панели).

## Реализация (технические аспекты)

### Модель данных

```python
class Subscription(Base):
    id: int (PK)
    user_id: int (FK → User.id, unique)
    plan: str (enum: "trial", "basic", "advanced")
    status: str (enum: "active", "expired", "cancelled")
    start_date: datetime
    end_date: datetime
    auto_renew: bool (default=True)
    created_at: datetime
    updated_at: datetime
```

### Лимиты по тирам

```python
PLAN_LIMITS = {
    "trial":   {"max_products": 10,  "fuzzy_search": False, "price_alerts": False, "export": False},
    "basic":   {"max_products": 100, "fuzzy_search": False, "price_alerts": False, "export": False},
    "advanced": {"max_products": 500, "fuzzy_search": True,  "price_alerts": True,  "export": True},
}
```

### Определение тарифа

```python
def get_user_plan(user) -> str:
    sub = get_active_subscription(user.id)
    if not sub or sub.end_date < now():
        return "none"
    return sub.plan

def get_plan_limits(plan: str) -> dict:
    return PLAN_LIMITS.get(plan, PLAN_LIMITS["trial"])
```

### Проверка доступа к умному поиску

```python
def can_use_fuzzy_search(user) -> bool:
    plan = get_user_plan(user)
    return PLAN_LIMITS[plan]["fuzzy_search"]
```

### Логика поиска

```python
async def search_products(query: str, user) -> list[Product]:
    plan = get_user_plan(user)
    limits = get_plan_limits(plan)

    # 1. Точное совпадение (всегда)
    exact = await exact_search(query)

    # 2. Частичное совпадение (всегда)
    partial = await partial_search(query)

    results = merge(exact, partial)

    # 3. Умный поиск (только для продвинутого)
    if limits["fuzzy_search"]:
        fuzzy = await fuzzy_search(query)  # pg_trgm
        results = merge(results, fuzzy)

    return sort_by_price(results)
```

### Проверка лимита отслеживания

```python
async def can_track_product(user, product_id) -> bool:
    plan = get_user_plan(user)
    limits = get_plan_limits(plan)
    tracked_count = await count_tracked_products(user.id)
    return tracked_count < limits["max_products"]
```

### Уведомления о снижении цен

```python
async def check_price_drop(product_id: int, old_price: Decimal, new_price: Decimal):
    if new_price >= old_price:
        return
    subscribers = await get_subscribers_for_product(product_id, plan="advanced")
    for user in subscribers:
        await send_notification(user, f"Цена на {product.name} снизилась: {old_price} → {new_price}")
```

## FAQ

**Что будет после 7 дней бесплатно?**
Поиск блокируется. Нужно оформить подписку. Если оформить в течение 7 дней после окончания — скидка 20%.

**Что такое умный поиск?**
Находит товары при опечатках, другой раскладке, транслитерации. «iphon 13 dsplay» → «Дисплей iPhone 13». «экран Самсунг Галакси С22» → «Дисплей Samsung Galaxy S22 Ultra». Только на продвинутом.

**Зачем уведомления о цене?**
Мастер постоянно закупает одни и те же детали. Если цена упала — он должен узнать об этом сразу, а не через неделю, когда зайдёт на сайт. Только на продвинутом.

**Что будет с данными после отмены подписки?**
Все данные (история поиска, отслеживаемые товары) сохраняются. При повторном оформлении — всё восстанавливается.
